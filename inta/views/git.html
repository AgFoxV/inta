第一节：版本管理介绍
版本管理系统简而言之就是记录文件或文件夹的变化，以便一段时间过后找到对应每个变化的文件或文件夹，1.直接粘贴复制，就是版本管理 2.本地版本管理系统，但不能合作开发 3.分布版本管理系统
第二节：Git基础
1.三个状态，committed 2 modified 3 staged 2.三个状态对应的三个工作路径，1.working directory staging area .git directoyr(repository) 3.Git directory clone will copy 4.working路径是项目版本的一个独立的checkout（35）。staging area是一个文件，在你的Git directory 下，保存下一个版本，将会改变的部分 一、在workingdirectory中改变文件，二。adding snapshots of them to your stagin area 三、提交成为git的一个部分
第三节：设置Git
1./etc/gitconfig -- system 2. gitonfig /.config /git/config --global
第四节：Git的基本命令
主要讲述Git的一些基本命令，如git，git commit 加a 可以直接跳过，进行提交，git rm 是在stage area 删除文件。加cached就是只在working directory下remove文件（57） git mv 这是重新命名文件 git log看history -p -2(59)git commit --amend git reset git tag命令 在说一个alias
