第一节：版本管理介绍
版本管理系统简而言之就是记录文件或文件夹的变化，以便一段时间过后找到对应每个变化的文件或文件夹。
第二节：Git基础使用
如何最基本的使用git进行版本管理
git init 开始管理你的项目和文件夹
git add 文件名 把文件加到暂存台
git commit -m “描述信息”
git checkout 描述符 切换到任何快照
git checkout master 返回最新快照
git tag -a 版本号 -m '版本信息' 提交版本信息
git checkout 版本号 切换相应的版本
每次提交都会产生一个快照。git正式通过快照存储文件的相关信息。 1.三个状态，committed 2 modified 3 staged 2.三个状态对应的三个工作路径，1.working directory staging area .git directoyr(repository) 3.Git directory clone will copy 4.working路径是项目版本的一个独立的checkout（35）。staging area是一个文件，在你的Git directory 下，保存下一个版本，将会改变的部分 一、在workingdirectory中改变文件，二。adding snapshots of them to your stagin area 三、提交成为git的一个部分
第三节：git状态

git status 显示当前git的状态
git中文件的两种状态：已跟踪(tracked)和未跟踪(untracked)
git rm --cached 文件 把已跟踪文件转为未跟踪文件 1./etc/gitconfig -- system 2. gitonfig /.config /git/config --global
第四节：暂存台操作

git diff 未跟踪文件和暂存台中文件的不同
git diff --staged 查看暂存台中的文件和最后一次提交的不同
git commit -a -m ‘描述信息’ 对于已在暂存台中的文件，更改后可以通过此命令直接提交。
git rm -f 文件名 从暂存台和当前目录中删除文件
git rm --cached 文件名 只在暂存台中删除

注：视频中讲解删除暂存台中文件时，视频中的话语是‘工作台’，应改为暂存台。

第五节：撤销更改和重复提交

git checkout -- 文件名 撤销更改
git commit --amend -m '描述信息' 覆盖提交
第六节：日志信息

git log -p 展示每次提交的不同
--stat 简单展示git的状态
--pretty=参数 格式化日志信息
--since=时间 展示指定提交时间后的信息
--graph 可以和pretty=oneline一起使用展示分支和合并的信息
第七节：主分支
git分支有着git特有的运行机制，本节主要讲述git主分支(master)与快照的运行机制。
第八节：分支

git branch 分支名 创建分支
git checkout 分支名 切换分支
git log - - decorate 查看当前分支
git checkout -b 分支名 直接创建加切换
第九节：提交分支
git log - - oneline - - decorate - - graph - - all
我们可以通过此命令图表化查看分支信息及分支的提交情况
第十节：合并分支
合并分支分两种方式：
第一种是 不创建快照合并
在master分支最后一个快照，创建的分支。合并后只需把HEAD指针直接移到合并分支的最新提交处。
此方法官方称之为:Fast-Forward
第二种是 创建快照合并
在master一个快照创建分支后，继续对master分支中的文件进行更改提交，合并时，git会创建新的快照合并分支。
此方法官方称之为：Three-way
git merge 分支名 合并分支
git checkout -d 分支名 删除分支

那年程序员

一个人写了两个系统，一个是linux操作系统，一个git版本管理系统

Git由Linus Benedict Torvalds在2005年开发的，Linus是一个瑞典的计算机大神。生于1969年12月28日，在写这篇《那年程序员》时，正好是2016年12月28日，星期三。估计47岁Linus大神这辈子也不知道，远在遥远东方，中国，北京，西小口地铁站旁的一间被雾霾笼罩的破房子中，有人祝福他的生日。说到linus,当我们看到这个名字的时候，对于一个程序员来说就应该知道，他就是那个编写linux内核的人。如果搜索这个人的相关信息，一系列的铺天盖地的关于他的传奇信息就会随之而来，而git的出现，最早也是为了管理linux内核的版本，当然现在的是由Hamano Jun一个日本的黑客维护，曾经是黑客。现在在google工作，住在程序员的麦加--中关村加利福尼亚软件园。Linus说过，认识Hamano及让他维护git是Linus最大的成功。也不知道git是不是应该翻译成‘基特’。anyway,Git是目前最流行的版本管理系统。为这个庞大的猿类世界提供了很多优秀的开源代码。生日快乐，linus！
