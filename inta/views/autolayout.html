AutoLayout课程概述

AutoLayout是IOS设备的UI界面的搭建接口，可以不写一行代码，直接拖拖拽拽完成UI界面的搭建

第一节：尺寸
本节介绍了IOS不同设备之间的尺寸。
retina屏和普通屏的区别
像素点和坐标点的映射
第二节：图像

讲述UIImageView中的图像
为什么会出现1x 2x 3x的图像
cpu 和 gpu
第三节：限制

限制是Autolayout的核心概念
如何为拖入的UI控件添加限制

添加限制的另一种方法，及其注意事项。

本节讲述如何对其UI控件
第六节：对齐(续）
对齐UI控价的另一种方法
第七节：固有尺寸
本节讲述UI控件的固有尺寸
UIView 没有固有尺寸
UISlider 只有固有宽度
UILabel, UIButton, UISwitch 和 Text Field既有宽度又有高度的固有尺寸
Text View 和 Image View 有内容就有固有尺寸
第八节：优先级
限制之间存在优先级，我们可以为其设置优先级

那年程序员

省时，省力，省心，不就是花点时间学习下儿吗

2011年，苹果公司提出了针对iOS的不同设备，使UI构件自动适应不同的屏幕界面的，且在不同的屏幕界面完成不同的布局。也就是我们说的Autolayout。
2011年7月6日，首次提出UI构件的基本限制模式
2012年2月16日，更改Autolayout代码错误
2012年9月19日，增加到iOS库，在WWDC大会上首次提出
2013年9月18日，在xcode5中，增加对Autolayout的描述
2015年9月16日，Autolayout增加了很多新的特性，功能，和插件。
2015年12月10日，最新最权威的Autolayout使用说明出版。
